Clickjacking:
- Interface-based attack which a user is tricked into clicking action on hidden website by clicking on other content in decoy web
- This technique is possible whenever website can be framed.
- Different from CSRF:
    + Clickjacking need click, CSRF don't need
    + Clickjacking trick user to click hidden UI, CSRF fake all the request
    + Clickjacking see a decoy web, CSRF don't see anything
- Hidden iframe in real web -> User is tricked to click -> Valid token include in request -> Send request with valid token + real domain -> Server accept action

Construct a basic clickjacking attack:
- Must fix the click div in same position as what you want victim click

Clickjacking with prefilled form input:
- Can prefill form input by putting param in url

Frame busting scripts:
- This is technique for defense against clickjacking, based upon restricting the framing capability for websites.
    + Common client-side protection is use frame busting or frame breaking scripts
    + Can be implement Javascript add-on or extensions such as Noscript
- Often perform some or all following behaviors:
    + Check and enforce that the current application window is the main or top window,
    + Make all frames visible
    + Prevent clicking on visible frames
    + Intercept and flag potential clickjacking attacks to the user.
- This technique depend on browser, platform specific and can be bypass 
- Effective way for attacker is to use HTML5 iframe sandbox attribute
    + When set with allow-forms or allow-scripts values and allow-top-navigation value is omitted, frame busting can't check whether or not it is the top window
    <iframe id="victim_website" src="https://victim-website.com" sandbox="allow-forms"></iframe>

Combining clickjacking with a DOM XSS attack:
- Historically, this used to perform behaviors such as boosting likes on Fb page
- However, true potency is revealed when used for DOM XSS attack

Multistep clickjacking:
- In some case, manipulation of input to target web may need multiple actions.
    Example: Buying product need to put it in shopping basket and checkout.
- These actions can implemented by using multiple divisions or iframes

How to prevent clickjacking attacks:
- 2 mechanism for server-side protection: X-Frame-Options, Content Security Policy (CSP)
- X-Frame-Options:
    + Orinally introduced as an unofficial response header in Internet Explorer 8 (IE8) adn rapidly adopted within other browsers.
    + Provide website owneer control over use of iframe object
        X-Frame-Options: deny
        X-Frame-Options: sameorigin
        X-Frame-Options: allow-from https://test.com
    + Not implemented consistently across browsers (allow-from not supported in Chrome 76, Safari 12 for example)
    + When applied combine with CSP, provide effective protection
- Content Security Policy:
    + Detection and prevention mechanism that provides mitigation against attacks such as XSS and clickjacking.
    + Usually implemented as a return header of form:
        Content-Security-Policy: policy
        + where policy is a string of policy directives separated by semicolons.
    + The recommended clickjacking protection is to incorporate the 'frame-ancestors' directive in the application's Content Security Policy
        Content-Security-Policy: frame-ancestors 'none';    (Same as X-Frame-Options deny)
        Content-Security-Policy: frame-ancestors 'self';    (same as sameorigin)
        Content-Security-Policy: frame-ancestors normal-website.com;
