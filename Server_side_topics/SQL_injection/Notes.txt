- Retrieving hidden data
- Subverting application logic
- Blind SQL injection vulnerabilities
- Second-order SQL injection (Useful when register not filter special char)
- Examining the database:
+ Some database have different: Syntax for string concatenation, Comments, Batched (or stacked) queries, Platform-specific APIs, Error messages.
+ First step: query version of database, database tables exist, and the columns they contain
    SELECT * FROM v$version (Oracle)
    SELECT * FROM information_schema.tables
+ https://portswigger.net/web-security/sql-injection/cheat-sheet
- UNION attack:
+ Determining the number of columns required
+ Finding columns with a useful data type
+ Retrieving multiple values within a single column
- Blind SQL injection:
+ Triggering conditional responses
    + triggering conditional errors
+ Extracting sensitive data via verbose SQL error messages
+ Triggering time delays
+ Out-of-band (OAST) techniques

Oracle db:
- Every 'SELECT' query must use the 'FROM' keyword and specify a valid table
- Has built-in table on Oracle called 'dual' (' UNION SELECT NULL FROM DUAL--)
- SELECT BANNER, NULL FROM v$version
- Has 'all_tables' return 'TABLE_CATALOG', 'TABLE_SCHEMA', 'TABLE_NAME', 'DATA_TYPE'
      'all_tab_columns' return like above plus 'COLUMN_NAME'

MySQL db:
- Comment by '--' must have at least 1 space after it '-- ', other db don't need

Other except Oracle:
- 'information_schema.tables' return 'TABLE_CATALOG', 'TABLE_SCHEMA', 'TABLE_NAME', 'TABLE_TYPE'
- 'information_schema.columns' return like above plus 'COLUMN_NAME'

Oracle --> v$version
PostgreSQL --> version()
Microsoft SQL Server --> @@version
MySQL --> version() and @@version
