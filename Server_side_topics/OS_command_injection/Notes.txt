02_Blind_OS_command_injection_with_time_delays
- Find api that could use shell code -> Try execute shell code by add injection to param
- Description:
+ We know that command format look like:
    mail -s "This site is great" -aFrom:peter@normal-user.net feedback@vulnerable-website.com
+ So in this case, we have a lots way to execute shell:
    ";ping+-c+10+127.0.0.1;" in msg or subject (cuz these param may have quote, need to go out of it to execute shell)
    "||ping+-c+10+127.0.0.1||"
    ||ping+-c+10+127.0.0.1|| in mail (cond1 || cond2 || cond3, with this first cond1 would fail and cond2 execute)
+ It will delay server response by 10s
+ If it work, we can sure that server have this type of vuln

Ways of injecting OS commands:
- Work in window and Unix-based systems: &, &&, |, ||
- Work in Unix-based systems: ';', Newline (0x0a or \n),
+ perform inline execution: `, $()

How to prevent OS command injection attacks:
- Validating against a whitelist of permitted values.
- Validating that the input is a number.
- Validating that the input contains only alphanumeric characters, no other syntax or whitespace.
