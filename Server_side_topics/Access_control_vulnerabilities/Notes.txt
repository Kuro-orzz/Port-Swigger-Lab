Vertical access controls:
- Control user based on roles (admin, user,..)
- Apply Least Privilege Principle
Horizontal access controls:
- Authorization based on ownership, restrict access to resources to specific users
- Advoid user access other users resources 
Context-dependent access controls:
- Restrict access to functionality and resources based upon the state of the application or the user's interaction with it.
- Prevent a user performing actions in the wrong order.
- Ex: prevent users from modifying the contents of their shopping cart after they have made payment.

Vertical privilege escalation:
- Unprotected functionality
- Parameter-based access control methods
- Broken access control resulting from platform misconfiguration
- Broken access control resulting from URL-matching discrepancies
+ In Spring 5.3, useSuffixPatternMatch option is enabled by default, so /admin/deleteUser.anything would still match the /admin/deleteUser pattern
+ In some system, /admin/deleteUser and /admin/deleteUser/ is treated as distinct endpoint
